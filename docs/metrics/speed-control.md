# 배속 선택 (Speed Control)

**상태**: 출시됨 (2026-04-04)
**기준선 측정일**: 2026-04-04 (출시 당일 GA4 30일 데이터)

## 배경

설문 응답에서 3명이 독립적으로 속도 조절을 요청:
- "속도조절이 되면 좋겠습니다" (학부모)
- "스피드가 빨라지는 과정" (교사)
- "박자가 조금 느린 버전도 있으면 좋을 것 같습니다" (교사)

BPM 182 고정이라 초등 저학년에게는 빠르고, 고학년에게는 쉬울 수 있음.

## 가설

> 속도 조절이 가능하면, 난이도 부담으로 이탈하던 사용자가 게임을 완료까지 플레이하고,
> 쉽다고 느끼던 사용자는 빠른 속도로 재도전하여 재도전율이 상승할 것이다.

## 기준선 (출시 전 30일)

| 지표 | 값 | 소스 |
|------|-----|------|
| 게임 완료율 | 59.3% (9,025 / 15,218) | game_complete / game_start |
| 재도전율 | 51.4% (4,637 / 9,025) | game_replay / game_complete |
| 평균 완료 라운드 | 3.25 (49,462 / 15,218) | round_complete / game_start |

## 목표 지표 (North Star)

**`speed_select` 기능 채택률** = speed_select 이벤트 고유 사용자 / game_start 고유 사용자

- 기준선: 0% (미존재)
- 출시 직후 (1일): 81명 / ~500 DAU = ~16%
- 목표 (2주 후): **15%+** 안정적 채택

## 보조 지표

| 지표 | 기준선 | 기대 방향 | 설명 |
|------|--------|-----------|------|
| 게임 완료율 | 59.3% | 상승 | 느린 속도로 완료 가능 |
| 재도전율 | 51.4% | 상승 | 빠른 속도로 재도전 |
| 평균 세션 체류시간 | 6분 43초 | 유지/상승 | 느린 속도 = 더 오래 체류 |
| **부작용 감시**: 이탈률 | 22.8% | 변동 없음 | UI 복잡성 증가로 이탈 증가 여부 |

## 추적 이벤트

| 이벤트 | 상태 | 파라미터 | 용도 |
|--------|------|----------|------|
| `speed_select` | 기존 | `challenge_id`, `speed` | 속도 선택 추적 |
| `game_start` | 기존 | `challenge_id` | 퍼널 시작점 |
| `game_complete` | 기존 | `challenge_id` | 퍼널 완료점 |
| `game_replay` | 기존 | `challenge_id` | 재도전 추적 |

### 분석 쿼리 포인트

- 속도별 완료율: `speed_select.speed` → 해당 세션의 `game_complete` 여부
- 속도별 재도전율: 완료 후 `game_replay` 비율을 속도별로 비교
- 가장 많이 선택되는 속도 분포

## 성공 기준

| 시점 | 기준 | 판단 |
|------|------|------|
| 1주 후 | 채택률 10%+ | 기능 발견성 OK |

| 2주 후 | 채택률 5% 미만 | UI 진입점 개선 필요 |

## 모니터링

- **1주차**: 일별 speed_select 이벤트 수 + 속도 분포 확인
- **2주차**: 완료율/재도전율 비교 (출시 전 vs 후)
- **이후**: 월간 리포트에 포함
