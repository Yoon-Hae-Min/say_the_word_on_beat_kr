# 배속 선택 (Speed Control)

**상태**: 출시됨 (2026-04-04)
**기준선 측정일**: 2026-04-04 (출시 당일 GA4 30일 데이터)

## 배경

설문 응답에서 3명이 독립적으로 속도 조절을 요청:
- "속도조절이 되면 좋겠습니다" (학부모)
- "스피드가 빨라지는 과정" (교사)
- "박자가 조금 느린 버전도 있으면 좋을 것 같습니다" (교사)

BPM 182 고정이라 초등 저학년에게는 빠르고, 고학년에게는 쉬울 수 있음.

## 가설

> 속도 조절이 가능하면, 난이도 부담으로 이탈하던 사용자가 게임을 완료까지 플레이하고,
> 쉽다고 느끼던 사용자는 빠른 속도로 재도전하여 재도전율이 상승할 것이다.

## 기준선 (출시 전 30일)

| 지표 | 값 | 소스 |
|------|-----|------|
| 게임 완료율 | 59.3% (9,025 / 15,218) | game_complete / game_start |
| 재도전율 | 51.4% (4,637 / 9,025) | game_replay / game_complete |
| 평균 완료 라운드 | 3.25 (49,462 / 15,218) | round_complete / game_start |

## 목표 지표 (North Star)

**`speed_select` 기능 채택률** = speed_select 이벤트 고유 사용자 / game_start 고유 사용자

- 기준선: 0% (미존재)
- 출시 직후 (1일): 81명 / ~500 DAU = ~16%
- 목표 (2주 후): **15%+** 안정적 채택

## 추적 이벤트

| `speed_select` | 기존 | `challenge_id`, `speed` | 속도 선택 추적 |

### 분석 쿼리 포인트

- 속도별 완료율: `speed_select.speed` → 해당 세션의 `game_complete` 여부
- 속도별 재도전율: 완료 후 `game_replay` 비율을 속도별로 비교
- 가장 많이 선택되는 속도 분포

## 결과 (출시 후 2일: 2026-04-04 ~ 04-05)

### 트래픽 컨텍스트

배속 기능은 바이럴 트래픽 소진 시점에 출시되었다. DAU가 03-30 피크(837명) → 04-05(147명)으로 82% 감소한 상태이므로, **출시 전/후 절대 수치 비교는 사용자 구성 변화를 반영하지 못한다.** 아래 수치는 이 맥락에서 해석해야 한다.

| 날짜 | DAU | game_start | game_complete | game_replay |
|------|-----|------------|---------------|-------------|
| 04-02 (출시 전) | 501 | 2,724 | 1,613 (59.2%) | 770 (47.7%) |
| 04-03 (출시 전) | 408 | 2,944 | 1,766 (60.0%) | 1,089 (61.7%) |
| **04-04 (출시일)** | **124** | **373** | **196 (52.5%)** | **86 (43.9%)** |
| **04-05** | **147** | **473** | **245 (51.8%)** | **82 (33.5%)** |

### 채택률 (North Star)

| 지표 | 값 | 판정 |
|------|-----|------|
| speed_select 이벤트 | 845회 (178명) | |
| game_start 이벤트 | 846회 (179명) | |
| **채택률** | **178/179 = 99.4%** | 목표 15% 대비 **6.6배 초과** |
| play_page_view → speed_select | 178/192 = 92.7% | 플레이 페이지 방문자 대부분 사용 |
| 인당 평균 speed_select | 845/178 = 4.75회 | 매 게임마다 속도 선택 |

**해석**: 채택률 99.4%는 기능이 게임 시작 플로우에 자연스럽게 통합되어 있음을 의미한다. 사용자가 "능동적으로 발견"한 것이 아니라 **플로우 상 반드시 거치는 UI**에 배치된 결과다. 이는 발견성(discoverability) 측면에서는 완벽하지만, "기본값(1x)이 아닌 속도를 선택한 비율"을 알아야 진정한 채택률을 측정할 수 있다.

### 데이터 한계

1. **속도 분포 조회 불가**: `speed` 파라미터가 GA4 커스텀 디멘션으로 등록되지 않아, 어떤 속도(0.5x, 0.75x, 1x, 1.25x, 1.5x)가 얼마나 선택되었는지 알 수 없다. → **GA4 관리자에서 커스텀 디멘션 등록 필요**
2. **샘플 크기**: 출시 후 2일, 약 180명 — 통계적으로 유의미한 결론을 내리기엔 부족
3. **트래픽 구성 변화**: 바이럴 유입(초등 교사/학부모, 수업 중 사용) → 자연 유입(개인)으로 전환 중이므로 사용자 행동 패턴 자체가 다를 수 있음

### 후속 액션

1. **GA4 커스텀 디멘션 등록** — `speed` 파라미터를 이벤트 스코프 디멘션으로 등록하여 속도별 분포 추적 가능하게
2. **1주 후 재측정** — DAU가 안정화된 시점(바이럴 영향 소진)에서 완료율/재도전율 재비교
3. **속도별 완료율 분석** — 커스텀 디멘션 등록 후, 느린 속도 사용자의 완료율이 높은지 검증 (가설의 핵심)
4. **기본값 선택 비율 분리** — 1x를 그대로 두고 시작한 비율 vs 능동적으로 속도를 변경한 비율 구분
