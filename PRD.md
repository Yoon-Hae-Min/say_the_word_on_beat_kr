📑 프로젝트 기획서: 단어리듬게임 (가제)

1. 프로젝트 개요
   서비스 명: 단어리듬게임

서비스 정의: 웹에서 누구나 쉽게 'Say The Word On Beat' 챌린지를 만들고 공유하는 리듬 퀴즈 플랫폼.

핵심 타겟: 숏폼 트렌드에 민감한 Z세대, 리듬 게임과 퀴즈를 즐기는 사용자.

플랫폼: 모바일 퍼스트 웹 (PC에서도 이용 가능).

개발 목표: Next.js와 Supabase를 활용하여 '사용자 생성 콘텐츠(UGC)' 흐름을 완벽하게 구현한다.

2. 전체 유저 플로우 (User Flow)
   사용자의 행동 흐름을 도식화했습니다.

코드 스니펫

flowchart TD
Landing[랜딩 페이지] --> Hero[히어로 섹션: 만들기 버튼]
Landing --> Feed[피드 섹션: 인기 챌린지 리스트]

    Hero --> StartModal[설정 모달 Open]

    subgraph Setup [초기 설정]
        StartModal --> Policy[정책 동의: 필수]
        Policy --> Visibility[공개/비공개 설정]
        Visibility --> GoEditor[에디터 진입]
    end

    GoEditor --> MakerPage[메이커 페이지]

    subgraph Making [제작 과정]
        MakerPage --> InputTitle[상단: 챌린지 제목 입력]
        MakerPage --> Upload[우측: 이미지 업로드 & 이름 입력]
        Upload --> Mapping[중앙: 8비트 슬롯에 이미지 배치]
        Mapping --> Complete[최종 생성 버튼]
    end

    Complete --> SuccessModal[완료 모달: 링크 공유]

    Feed --> PlayPage[플레이 페이지 진입]
    SuccessModal --> PlayPage

3. 페이지별 상세 기획
   A. 랜딩 페이지 (Landing)
   서비스의 대문이자, 다른 사람의 작품을 구경하는 곳입니다.

히어로 섹션 (Hero)

강렬한 타이포그래피와 함께 "나만의 단어리듬게임 만들기" 버튼(CTA) 배치.

버튼 클릭 시 [설정 모달] 호출.

피드 섹션 (Feed)

노출 기준: is_public: true인 챌린지만 노출.

정렬 기준: 인기순 (조회수/플레이 횟수 높은 순) 내림차순 정렬.

UI: 카드 리스트 형태 (썸네일, 제목, 조회수). 클릭 시 플레이 페이지로 이동.

B. 설정 모달 (Start Modal)
에디터 진입 전, 필수 절차를 거치는 단계입니다.

정책 동의: "부적절한 콘텐츠 게시 시 삭제될 수 있음" 체크박스 (동의 안 하면 진행 불가).

공개 여부 설정:

공개: 피드에 노출됨.

비공개: 링크가 있는 사람만 접근 가능.

확인: [만들러 가기] 버튼 클릭 시 메이커 페이지로 이동.

C. 메이커 페이지 (Maker / Editor)
실제 챌린지를 제작하는 작업 공간입니다.

상단 영역 (Header):

챌린지 제목 입력: "어떤 챌린지인가요?" (예: 아이돌 노래 맞추기).

우측 패널 (Resources):

이미지 업로드: 로컬 파일 선택.

리소스 리스트: 업로드된 이미지 썸네일 나열.

이름 입력: 선택한 이미지의 정답 단어 입력 (플레이 시 정답 확인용).

중앙 영역 (Stage):

8개의 슬롯: 비트 박자에 대응하는 8개의 빈 칸.

매핑 로직: [우측 이미지 선택] → [중앙 슬롯 클릭] = 이미지 배치.

라운드 제어: 총 5라운드 구성 (Previous / Next 버튼).

하단 영역 (Footer):

생성하기 버튼: 모든 데이터 저장 후 [완료 모달] 호출.

D. 완료 모달 (Success Modal)
결과: 생성된 챌린지 썸네일 표시.

액션:

[링크 복사]: 클립보드에 URL 복사.

[바로 플레이]: 내가 만든 거 해보기.

E. 플레이 페이지 (Player)
게임이 진행되는 실제 화면입니다.

구성:

화면 중앙에 4x2(또는 적절한 그리드) 형태의 이미지 슬롯 8개 배치.

상단에 현재 라운드(Round 1/5) 표시.

게임 로직 (BGM & Timing):

Phase 1 (Preview): 인트로 음악 재생. 8개 이미지가 모두 보임 (사용자가 미리 눈으로 익힘).

Phase 2 (On Beat): 비트 시작. 박자에 맞춰 1번 칸부터 8번 칸까지 순차적으로 Border Highlight(포커스) 이동.

Interaction: 사용자는 포커스된 타이밍에 맞춰 단어를 외침 (시스템적 판정은 없으며, 사용자 스스로 즐기는 방식).

데이터 처리:

페이지 로드 시 해당 챌린지의 조회수(View Count) +1 증가 (인기순 정렬을 위함).
