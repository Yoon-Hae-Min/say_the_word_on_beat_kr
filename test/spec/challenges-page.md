# 챌린지 리스트 페이지 (/challenges)

## TC-1: 페이지 로드 & 기본 렌더링

1. `/challenges` 페이지에 접속한다
2. "공개된 챌린지" 제목이 보이는지 확인한다
3. "총 N개의 챌린지" 문구에서 N이 1 이상인지 확인한다
4. 챌린지 카드가 1개 이상, 12개 이하로 표시되는지 확인한다

## TC-2: 헤더 네비게이션

1. `/challenges` 페이지에서 "홈으로" 버튼을 클릭한다 → 홈(`/`)으로 이동하는지 확인한다
2. 뒤로가기 후 "만들기" 버튼을 클릭한다 → `/maker` 페이지로 이동하는지 확인한다
3. 뒤로가기 후 "내 챌린지" 버튼을 클릭한다 → `/my` 페이지로 이동하는지 확인한다

## TC-3: 챌린지 카드 클릭

1. `/challenges` 페이지에서 첫 번째 챌린지 카드를 클릭한다
2. `/play/{챌린지ID}` 형태의 플레이 페이지로 이동하는지 확인한다

## TC-4: 정렬 — 전체 옵션 순회

기본 정렬은 "추천순"이다. 아래 5개 옵션을 순서대로 선택하며 검증한다.

1. "조회순"을 선택한다 → URL에 sort=views가 반영되고, 챌린지 카드가 1개 이상 표시되는지 확인한다
2. "이번주 인기"를 선택한다 → URL에 sort=views_week가 반영되고, 챌린지 카드가 1개 이상 표시되는지 확인한다
3. "이번달 인기"를 선택한다 → URL에 sort=views_month가 반영되고, 챌린지 카드가 1개 이상 표시되는지 확인한다
4. "최신순"을 선택한다 → URL에 sort=latest가 반영되고, 챌린지 카드가 1개 이상 표시되는지 확인한다
5. "추천순"을 선택한다 → URL이 기본 상태로 돌아가고, 챌린지 카드가 1개 이상 표시되는지 확인한다

> 참고: 기본값(sort=recommended, page=1)은 URL 파라미터에서 생략된다.

## TC-5: 페이지네이션

1. `/challenges` 페이지에서 "다음" 버튼이 존재하는지 확인한다
2. "다음"을 클릭한다 → URL에 page=2가 반영되는지 확인한다
3. 2페이지에 챌린지 카드가 1개 이상 표시되는지 확인한다
4. "이전"을 클릭한다 → 1페이지로 돌아가는지 확인한다
5. 페이지 번호 "3"을 클릭한다 → URL에 page=3이 반영되는지 확인한다

## TC-6: 정렬 변경 시 page 리셋

1. "다음"을 클릭하여 2페이지로 이동한다
2. 정렬을 "최신순"으로 변경한다
3. URL에 sort=latest가 반영되고, page가 1로 리셋되었는지 확인한다
